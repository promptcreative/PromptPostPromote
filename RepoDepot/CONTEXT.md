# CONTEXT

**PROJECT NAME:** PromptPostPromote / Astrobatching

**GITHUB URL:** https://github.com/promptcreative/PromptPostPromote

**PURPOSE:** Astrobatch promotion calendar and agency tools that calculate personalized content posting times from birth data using multiple astrological calendar systems (Vedic, Magi/PTI, Personal Transit), find overlapping "golden windows" (OMNI, Double GO, Good classifications), filter microtransits to background days, compute Micro Bird precision posting times, and export via ICS calendar subscriptions and Publer API. Supports agency-style multi-client management with a client portal.

**CURRENT STATUS:** Functional and actively used. All core systems operational:
- Five-engine calendar generation pipeline (PTI, Vedic, Personal, Bird Batch, Combined)
- Power Days classification (OMNI, Double GO, Good) with background day filtering
- Bird Batch on background days, Yogi Point, Part of Fortune microtransit layers
- Micro Bird precision posting windows (microtransit + bird batch overlap)
- ICS calendar subscription feeds (10+ feed types, permanent token-based URLs)
- Publer integration for pushing Micro Bird events as scheduled drafts
- Agency multi-client management with full pipeline per client
- Client portal (role-based auth: admin / editor / client / user)
- Manual calendar input system (editors enter Magi/PTI + Vedic data by hand)
- DB schema fully migrated (role column added March 2026)

**ARCHITECTURE:** Monolithic Python Flask application with blueprint-based route organization. PostgreSQL (Neon-hosted) for persistence. Separate modules for pages, API, clients, power days, publer, ICS feeds, microtransits, calendars, and downloads. Astrology engines in core/ and microtransits/ directories. Jinja2 templating with vanilla JS frontend. Session-based email auth with four-role system (admin/editor/client/user).

**TECH STACK:** Python 3.11, Flask, Flask-SQLAlchemy, PostgreSQL, SQLAlchemy, Requests, Pandas, pyswisseph, ephem, astral, pytz, icalendar, tabulate, gunicorn, psycopg2-binary

**USER FLOW:**
- **Regular user:** Signs up → enters birth data on profile setup → clicks "Generate my calendars" on dashboard → views Power Days (OMNI/Double GO/Good/Bird Batch/Micro Bird) → subscribes to ICS calendar feeds → optionally pushes Micro Bird events to Publer.
- **Admin:** Logs in with admin email → manages clients (CRUD) → generates calendars per client → views client results (full pipeline) → shares ICS feed URLs with clients.
- **Editor:** Logs in → goes to Calendar Input → enters manual Magi/PTI and Vedic classifications by date for the current month.
- **Client:** Logs in with email matching a Client record → lands on client portal dashboard → views their Power Days, Calendar Feeds (read-only, no regeneration). Calendar must be generated by admin first.

**HOW TO RUN:** Application runs from the `rebuild/` directory. Set environment variables: `DATABASE_URL`, `ADMIN_EMAIL`, `PUBLER_API_KEY`, `PUBLER_WORKSPACE_ID`, `SESSION_SECRET`. Run with `cd rebuild && python app.py`.

**KEY FILES:**
- `rebuild/app.py` — Flask app setup, DB config, blueprint registration
- `rebuild/database/models.py` — SQLAlchemy models (UserProfile, CalendarData, Client, Settings, SubscriptionToken, ManualCalendarEntry)
- `rebuild/database/manager.py` — Database manager for calendar data persistence and subscription token management
- `rebuild/routes/auth.py` — Email-based auth, role detection (admin/editor/client/user), profile management
- `rebuild/routes/pages.py` — Page routes (dashboards, clients, calendar feeds, power days, client portal)
- `rebuild/routes/clients.py` — Client CRUD + calendar generation/retrieval API (admin-only)
- `rebuild/routes/power_days.py` — Power Days API (classified days, filtered microtransits, bird batch, calendar feeds)
- `rebuild/routes/publer.py` — Publer integration (push microbird, push generic)
- `rebuild/routes/ics_feeds.py` — ICS calendar subscription feeds (10+ types, all + background-filtered)
- `rebuild/routes/manual_calendar.py` — Manual calendar entry API for editors/admin
- `rebuild/helpers/utils.py` — `get_effective_user_id()`, JSON serialization, date range utilities
- `rebuild/helpers/dashboard.py` — Core dashboard generation logic (runs all 5 engines)
- `rebuild/core/` — Astrology engines (PTI/Magi, Vedic, Personal Transit, Combined Calendar)
- `rebuild/microtransits/` — Yogi Point (yp.py), Part of Fortune (wb1.py), Western/Vedic bridge calculators
- `rebuild/templates/` — Jinja2 templates (landing, login, dashboards, power days, calendar feeds, client portal)

**DEPENDENCIES:**

- Publer API — Social media draft post creation from Micro Bird events
- Swiss Ephemeris (pyswisseph) — Planetary position calculations for all astrology engines
